# ADR Journal

> Журнал Architecture Decision Records для Artificial Life Engine.

---

# Призначення

Цей каталог містить ADR — документи, які фіксують причини важливих архітектурних рішень.

ADR не описує повну механіку. Механіка після прийняття рішення повинна бути описана у відповідному Canon-документі.

---

# Коли створювати ADR

ADR потрібен, якщо:

* змінюється фундаментальне правило світу;
* рішення вибирає один варіант із кількох Research-альтернатив;
* важлива ідея відхиляється й це треба зафіксувати;
* рішення впливає на майбутню реалізацію;
* без запису причини майбутній агент може відновити застаріле або відхилене знання.

---

# Коли ADR не потрібен

ADR не потрібен для:

* виправлення описок;
* синхронізації навігації з фактичною структурою;
* дрібних уточнень формулювання без зміни змісту;
* видалення застарілих згадок, які не мають документованої причини.

---

# Поточний стан

Прийнятих ADR ще немає.

Не створювати ADR-заглушки без описаної причини, контексту й прийнятого рішення.

Якщо причина для ADR не описана або рішення не прийняте, такий ADR не створюється.

---

# Формат майбутніх ADR

```text
ADR-0001-short-title.md
```

Кожен ADR повинен містити:

1. контекст;
2. проблему;
3. розглянуті варіанти;
4. прийняте рішення;
5. наслідки;
6. пов'язані документи.
